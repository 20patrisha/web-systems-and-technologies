Overview
  • Web development is the building and maintenance of websites.
  • In web development, we have frontend and backend.
  • Frontend or client-side is anything you see when you open a website.
  • Backend or server-side is responsible for storing and organizing data. This is where the frontend gets the data to be displayed on the website and also where it sends the data to be stored into the database. Any data related processes are done in the backend.
  • Frontend vs Backend
  • Frontend and Backend Diagram
    - frontend communicates with backend through HTTP requests and responses.
    - HTTP request is just requesting or sending data to backend.
    (eg. fetching data for frontend to display or sending the username and password from frontend to backend for verification).
    - HTTP response is the response from backend.
    (eg. data requested by the frontend or message if the user successfully logged in or anything)

HTML
  • HTML is the basic structure of the website.
  • This is where you put the elements that you want on your website. (eg. texts, tables, images, buttons, links, inputs)
  • Your HTML file is composed of HTML elements that looks like this.
    <button>Click me!</button>
  • Most of the html elements have opening and closing tags.
  • Put the HTML tags in the body tag not in head tag otherwise it will not be rendered.
  • Facebook with just HTML looks like this.
  • Make sure that you already know what your website would look like before starting to code.

CSS
  • CSS is the one that give styles to HTML elements. Styles include background color, text color, padding, margin etc.
  • Sample CSS code.
    body {
  background-color: lightblue;
}
h1 {
  color: white;
  text-align: center;
}
p {
  font-family: verdana;
  font-size: 20px;
}
  • Read here how CSS selects HTML elements to style. (through html element name, id or class)
  • Ways to insert CSS to HTML.
    1. External CSS - you create separate file for css (eg. styles.css) then link it in your HTML file inside the head tag. 
      <head>
  <link rel="stylesheet" href="styles.css"/>
</head>
    2. Internal CSS - you add CSS styles in style tag inside head tag. 
      <head>
  <style>
  body {
    background-color: linen;
  }

  h1 {
    color: maroon;
    margin-left: 40px;
  }
  </style>
</head>
    3. Inline CSS - adding style directly in an HTML element.
      4. <h1 style="color: blue; text-align: center"> This is a heading </h1>
  • Explore CSS here.
Bootstrap
  • Bootstrap is a CSS framework which is used for faster and easier web development
  • When using a CSS framework, you mostly don't need to do CSS unless you need custom styles.
  • Has ready-to-use classes for styling HTML elements. (eg. just adding "btn btn-primary" class to a button results to this)
  • Adding Bootstrap to HTML is easy. Just add this CSS link inside head tag.
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css" rel="stylesheet">
  • Some features of Bootstrap requires JavaScript (eg. modals). So you need to add a script link for it to work.
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/js/bootstrap.bundle.min.js"></script>

Git commands
  • You will be working on Git Bash which is a terminal for Git.
  • This will be installed along with Git.
  • To check if git is properly installed, run
    git --version
  • git config
    - Now let Git know who you are. This is important for version control systems, as each Git commit uses this information:
    Example:
    git config --global user.name "Owen Patrick Falculan"
    git config --global user.email owenpatrick.falculan@g.batstate-u.edu.ph"
  • git init
    - Makes the project folder a git repository. Changes inside this folder will be tracked by Git.
  • git add
    - This will add the files to staging environment.
    - Files in staging environment ar files that are ready to be commited.
    - You can add a single file (eg. git add "filename") or all files inside the folder (eg. git add .)
    Example:
    git add index.html
    git add .
  • git status
    - Checks the status of files
  • git commit
    - Commit is a save point or it is the version that we want to save.
    - It is a point in the project you can go back to if you find a bug.
    - When we commit, we should always include a message.
    Example:
    git commit -m "Add a back button"
  • git clone
    - Clones the repository and saves it to your local machine.
    Example:
    git clone git@github.com:owenfalculan/web-systems-and-technologies.git
  • git branch
    - Creates a branch or a new/separate version of the main repository.
    Example:
    git branch "add-about-me-section"
    git checkout "add-about-me-section"
    - Shortcut (create branch then switch):
    git checkout -b "add-about-me-section"
  • git push
    - Pushes or uploads your changes/branch/commit to remote repository (eg. GitHub and Gitlab).
  • git pull
    - Updates the local repository.
